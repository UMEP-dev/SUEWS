# suews-scm — a research-grade single-column model over the SUEWS framework

`suews-scm` couples the [SUEWS](https://suews.readthedocs.io) urban land-surface scheme (through SuPy) to a one-dimensional atmospheric column, turning the offline surface model into a closed land–atmosphere system for exploring large-scale urban climate questions: how cities reshape the boundary layer, how anthropogenic heat and surface modifications propagate into the air above, and how mitigation scenarios play out when the atmosphere is allowed to respond.

In offline SUEWS, air temperature, humidity and wind are *prescribed* from observations, so the atmosphere can never feel the surface. In the coupled SCM they are *prognostic*: the urban heat island, boundary-layer deepening over the city, and the air-temperature response to cool roofs all **emerge** from the physics rather than being imposed.

## Model components

- `suews_scm.column.ColumnModel` — multi-level boundary-layer column
  - Prognostic θ, q, u, v on a stretched grid (interfaces at the surface; fluxes and eddy diffusivities on interfaces, state at cell centres).
  - **Unstable conditions**: first-order non-local K-profile closure within the diagnosed boundary layer (Troen & Mahrt 1986), counter-gradient term and turbulent Prandtl number after Holtslag & Boville (1993).
  - **Stable / neutral conditions and the free atmosphere**: local Blackadar mixing-length closure with a Richardson-number stability function; default is the sharp cut-off form `f = (1 − Ri/0.2)²`, following the GABLS1 finding that long-tailed operational functions over-deepen the stable boundary layer (Cuxart et al. 2006). The long-tail form is available as an option.
  - **BL height**: bulk Richardson number (critical value 0.25) with the Vogelezang & Holtslag (1996) `100 u*²` shear term and a Troen–Mahrt convective thermal excess.
  - **Numerics**: backward-Euler implicit vertical diffusion in conservative flux form (column integrals conserved exactly up to the prescribed surface flux — tested to 1e-10), exact rotation for Coriolis, first-order upwind subsidence, optional nudging and prescribed clear-air radiative cooling.
  - **Surface boundary condition**: prescribed kinematic fluxes (coupled mode) or a prescribed surface temperature with Monin–Obukhov similarity (idealised mode; Paulson/Dyer unstable, log-linear stable).
- `suews_scm.slab.SlabCBL` — CLASS-style bulk convective boundary-layer model (zeroth-order jump, β = 0.2 entrainment closure, RK4), the modern descendant of the legacy SUEWS-CBL / BLUEWS slab scheme (Onomura et al. 2015) that still ships in `suews_phys_bluews.f95`.
- `suews_scm.native.run_coupled_native` — **the production backend**: the same coupled loop executed natively inside the SUEWS Fortran driver, reached through the Rust bridge in a single call
  - The column physics is ported line-for-line into `src/suews/src/suews_phys_scm.f95` (`module_phys_scm`); the coupled timestep loop is `SUEWS_cal_multitsteps_scm` in `suews_ctrl_driver.f95`, exposed as `suews_scm_multitsteps_c` (bridge C API) and `suews_bridge.run_suews_scm` (Python).
  - The Python `CoupledSCM` below remains the validated reference implementation; `tests/test_native.py` pins the two backends together (air-temperature agreement ≤ 0.15 K over a six-hour coupled window — residual differences come from the Python path's per-step state round-trips, not the physics).
  - Performance (measured, `benchmarks/run_native_vs_python.py`, M1 Max, debug-profile physics library in both backends): the full 5-day ventilated KCL episode takes **0.34 s native vs 187 s through the Python loop — a 555× speedup** (0.23 vs 130 ms per coupled step); skill against the observations is statistically indistinguishable (RMSE 3.41 K native vs 3.66 K Python, the small difference favouring the native loop's continuous state integration).
- `suews_scm.coupling.CoupledSCM` — the two-way coupler (reference implementation)
  - Each SUEWS step (5 min): column state at the forcing height → SUEWS forcing (T, RH, U); SUEWS QH, QE, u\* → kinematic flux boundary condition for the column substeps.
  - Radiation, precipitation and pressure remain prescribed from the forcing file.
  - **Wind**: nudged towards a log-profile anchored at the observed speed (`nudge_obs`, default — a 1-D model cannot generate the synoptic pressure gradient), or fully prognostic with a geostrophic wind (idealised experiments).
  - **Ventilation**: a fixed column over a city is not a closed system — advection replaces urban-heated air with upstream air on a time scale τ = L/U (~1 h for central London). The coupler therefore exchanges θ and q with a **background atmosphere generated by a companion rural coupled run** (itself quasi-closed, cooling at night through its own negative QH and clear-sky radiative cooling of 2 K day⁻¹). Without this term the urban column warms by several kelvin per day, which is exactly why the original BLUEWS scheme ran daytime-only with daily re-initialisation.

## Benchmarks (`benchmarks/`)

All metrics are regenerated by the scripts and stored as JSON under `benchmarks/results/`.

- **GABLS1 stable boundary layer** (`run_gabls1.py`) — the standard SBL intercomparison case (Beare et al. 2006 LES ensemble; Cuxart et al. 2006 SCM ensemble). With the sharp stability function the column gives h ≈ 168–198 m (stress/Ri-based), u\* ≈ 0.27 m s⁻¹ and a supergeostrophic low-level jet at ≈ 184 m after 9 h — all within the published LES ranges (quasi-steady depth ~150–250 m, u\* ~0.25–0.30 m s⁻¹). Switching to the long-tail function reproduces the documented over-deepening (stress-based h ≈ 378 m).
- **Convective boundary-layer growth** (`run_cbl.py`) — constant surface heat flux into linear stratification. Column h(t) within 4 % and slab within 0.3 % of the analytic entrainment growth law `h = √(h₀² + 2(1+2β)Ft/γ)` with β = 0.2 (Tennekes 1973; Garratt 1992). The column's flux-profile entrainment ratio (≈ 0.06) is weaker than the LES consensus (0.15–0.25; Sullivan et al. 1998) — a known limitation of first-order non-local schemes without an explicit entrainment closure (cf. Noh et al. 2003).
- **Coupled vs observations, KCL London** (`run_coupled_kcl.py`) — SUEWS spun up offline over 2012, then the coupled system (urban column ventilated by a companion rural column) run over the clearest summer window (22–27 July 2012). The modelled air temperature at 40 m is compared against the observation-based forcing the offline model would normally consume. Day 1 is excluded as column spin-up.

Unit and regression tests: `tests/` (tridiagonal solver vs dense reference, exact conservation, Monin–Obukhov log-law recovery, slab analytic solutions, GABLS1 ranges, coupled smoke test).

## Demo cases (`demos/`)

Coupled experiments over 22–27 July 2012, identical synoptic forcing:

1. `demo_urban_rural.py` — central London (KCL) vs a grassland configuration: the urban heat island and urban boundary-layer deepening as emergent properties.
2. `demo_anthropogenic_heat.py` — Järvi et al. (2011) QF coefficients × 0 / 1 / 2: how energy use warms the air a city breathes, and when (nocturnal accent).
3. `demo_cool_roofs.py` — building-facet albedo 0.12 → 0.55: a mitigation scenario where the atmosphere is allowed to respond (the offline model would only change the fluxes, never the air).

## Usage

```bash
# from the repository root, with the dev environment built (make dev)
cd scm
python -m pytest tests/            # unit + regression tests
python benchmarks/run_gabls1.py    # idealised benchmarks (no SUEWS needed)
python benchmarks/run_cbl.py
python benchmarks/run_coupled_kcl.py   # coupled benchmark (~6 min)
python demos/demo_urban_rural.py       # demo cases (~5 min each)
python demos/demo_anthropogenic_heat.py
python demos/demo_cool_roofs.py
python benchmarks/run_native_vs_python.py  # native vs reference backend
open site/index.html               # demo webpage with all results
```

Native backend, programmatically:

```python
from supy.data_model import SUEWSConfig
from suews_scm.native import run_coupled_native

config = SUEWSConfig.from_yaml("path/to/config.yml")
df_output, df_scm, state_json = run_coupled_native(
    config, df_forcing,             # SuPy-layout forcing
    state_json=spunup_state_json,   # optional, from a previous bridge run
    background=bg,                  # optional rural-companion ventilation
    city_length=15000.0,            # any SCM_PARAM_DEFAULTS key
)
```

## Honest limitations

- One column is one city: horizontal heterogeneity, sea breezes and synoptic changes enter only through the prescribed radiation/precipitation and the ventilation term.
- The ventilation background is *modelled* (rural companion run), not observed; the advective time scale uses a single city-length scale (15 km).
- Entrainment at the CBL top is under-represented (β_eff ≈ 0.06 vs 0.2 from LES); afternoon mixed-layer warming may be slightly underestimated.
- The wind profile in `nudge_obs` mode is constrained by observations; momentum is only fully prognostic in idealised mode.
- SUEWS's bulk building facet aggregates roofs and walls, so facet-albedo scenarios are upper bounds.
- Pressure–height conversions use a constant scale height per run; fine below 4 km.
